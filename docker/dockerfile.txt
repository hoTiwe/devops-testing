Задача: необходимо дать рекомендации/правки по улучшению/исправлению следующего Docker file# Используем базовый образ

FROM python

# Указываем рабочую директорию
WORKDIR /app

# Копируем файлы проекта
COPY . .

# Устанавливаем зависимости 
RUN pip install -r requirements.txt

# Экспонируем порт для Flask
EXPOSE 5000

# Указываем команду запуска приложения
CMD ["python", "app.py"]`

Рекомендации: 
1. FROM python без тега (при сборке будет менятся версия питона, при выходе новых версий)
Соответственно на продуктивах может не работать некоторый функционал. 
Так что ставим фиксированные версии, лучше alphine, более легковесные.
2. Копирование всех файлов проекта может сломать кешь слоя с pip install, зависимости будут ставится по новой.
3. pip install -r requirements.txt используем с флагами -no-cache-dir, дополнительный кеш от установки будет "мусором" в образе.
4. Не стоит документировать порт в образе. Если рассматриваем запуск в прод стенде, то лучше
настроить через ENV.
5. По умолчанию контейнер запускается из под рута, поэтому создаем не рутового пользователя, 
будем запускать из под него.
6. Создаем файл .dockerignore, чтобы не засорять образ.
